前端接口与占位符说明
生成日期：2026-03-30

对应源码文件：frontend/src/views/ProfileView.vue

1) API 地址占位符
- 行号：275
- 代码：const AI_CHAT_API_ENDPOINT = "https://api.placeholder.chat/3k9x2m7nq1";
- 作用：统一管理聊天接口地址。当前先用随机字符串风格的占位地址，后续接入真实大模型接口时只需要替换这一处。

2) 真实 API 接入预留点
- 行号：579
- 代码注释：// AI 调用接口位置：后续接入真实大模型时，替换为真实 API。
- 作用：明确告诉后续开发者，真正的后端调用应在这里落地，避免改动到无关逻辑。

3) API 调用示例占位
- 行号：580
- 代码注释：// 例如：await axios.post(AI_CHAT_API_ENDPOINT, payload)
- 作用：给出标准调用姿势（POST + payload），后续可以直接取消注释并替换为正式实现。

4) 请求参数 payload
- 行号：568
- 代码：const payload = { ... }
- 作用：把会话 ID、用户输入、画像快照、问卷答案、采集计数等上下文打包，作为未来大模型请求体。

5) AI 占位回复（随机标识）
- 行号：585
- 代码：return 占位回复(${randomTag}) ...
- 作用：在后端接口未接入时，保证聊天流程可用，并通过 randomTag 模拟“每次回复不同”的效果。

6) 采集数据中的接口占位字段
- 行号：544
- 代码：endpoint_placeholder: AI_CHAT_API_ENDPOINT,
- 作用：把当次对话使用的接口占位地址一并存档，便于后续训练数据回溯与来源追踪。

7) UI 中展示占位地址
- 行号：182
- 说明：聊天面板显示 API 占位地址。
- 作用：方便前端联调时快速确认当前是否仍在占位模式，避免误以为已经连到真实服务。

8) 聊天采集数据的本地存储 Key
- 行号：276
- 代码：const CHAT_DATA_STORAGE_KEY = "profile_ai_chat_training_data";
- 作用：统一 localStorage 键名，保证采集数据读写一致，避免出现多套键名导致的数据丢失。

9) 采集数据的本地读写位置
- 行号：514（读取）
- 行号：530（写入）
- 作用：实现聊天采集数据的持久化，刷新页面后仍可恢复，供后续大模型学习数据准备使用。
